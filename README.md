# Django Blog Project

## Описание

Этот проект представляет собой учебное Django-приложение (блог). Его основная цель — не только создать работающий блог, но и продемонстрировать современный и автоматизированный подход к разработке на Python.

## Инструменты

* uv — быстрый менеджер окружений и зависимостей.

* pre-commit — автоматический контроль качества кода.

* Makefile — единый интерфейс для автоматизации рутинных команд.

## Запуск проекта

1. Установка зависимостей

Убедитесь, что у вас установлен uv. Затем запустите команду для создания виртуального окружения и установки всех пакетов, зафиксированных в pyproject.toml:
```
uv sync
```

2. Запуск сервера

Для запуска локального сервера разработки используйте команду make, которая скрывает длинный вызов manage.py:
```
make run
```

Сервер будет доступен по адресу: http://127.0.0.1:8000/

## Качество кода и автоматизация

### Pre-commit хуки (локальная проверка)

Код проверяется на соответствие стандартам (линтеры, форматирование) перед каждой фиксацией (git commit). Это гарантирует, что в репозиторий попадает только чистый код.

Для ручного запуска всех проверок в любой момент используйте специальную команду из Makefile:
```
make pre-commit-check
```

### Настройка Git и защита ветки main

На ветке main настроены правила защиты для обеспечения стабильности и качества кода:

1. Запрет прямого пуша: Все изменения принимаются только через Pull Request (PR).

2. Блокировка Force Push: Запрещено принудительно перезаписывать историю.

3. Ограничение обновлений: Обновление ветки main ограничено для дополнительной безопасности.
